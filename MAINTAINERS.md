## Maintainer Responsibilities

All maintainers are expected to:

1. **Code Review**
   - Review pull requests in a timely manner
   - Ensure code quality and adherence to standards
   - Test changes before merging
   - Provide constructive feedback

2. **Issue Management**
   - Triage new issues
   - Label issues appropriately
   - Respond to questions and bug reports
   - Close resolved or duplicate issues

3. **Security**
   - Monitor security advisories
   - Review and address security vulnerabilities
   - Follow responsible disclosure practices
   - Keep dependencies up to date

4. **Documentation**
   - Maintain accurate documentation
   - Update guides and tutorials
   - Ensure README and contributing docs are current
   - Document architectural decisions

5. **Community**
   - Welcome new contributors
   - Enforce Code of Conduct
   - Foster inclusive environment
   - Represent project professionally

6. **Quality Assurance**
   - Ensure CI/CD pipelines pass
   - Run tests before merging
   - Verify builds are successful
   - Maintain code quality standards

## Scope of Maintainership

### Repository Areas

This repository encompasses:

- **Website Application**: Next.js frontend application
- **Documentation**: All .md files and guides
- **CI/CD**: GitHub Actions workflows
- **Testing**: Unit tests, E2E tests, accessibility tests
- **Infrastructure**: Deployment configuration, GitHub Pages setup

### Decision Authority

Maintainers have authority over:

- Pull request approval and merging
- Issue prioritization and labeling
- Minor technical decisions
- Code review standards
- Documentation updates

**Note**: Major architectural changes, security policies, and governance matters require approval from the Founder.

## Becoming a Maintainer

We welcome contributors who demonstrate:

- Consistent high-quality contributions
- Understanding of the codebase
- Positive community interactions
- Alignment with Free For Charity's mission

See [GOVERNANCE.md](./GOVERNANCE.md) for the full maintainer appointment process.

## Maintainer Status

### Active Maintainers

Active maintainers are those who:

- Have merged a PR in the last 6 months, OR
- Have reviewed/commented on issues/PRs in the last 6 months, OR
- Have communicated their continued involvement

### Emeritus Maintainers

Emeritus status is granted to past maintainers who:

- Have stepped down from active duties
- Made significant historical contributions
- Remain supportive of the project

**Current Emeritus Maintainers**: None (new project)
